## 1. 方案概述

### 1.1 方案A: 截断阈值优化

**来源**: `SCRAPER_FLOW_ANALYSIS_REPORT.md`

**核心思路**: 将截断阈值从8,000字符提高到150,000字符

```
当前流程 (不变):
  抓取 → 清洗 → 截断(8K) → 蒸馏 → 事实

优化后:
  抓取 → 清洗 → 截断(150K) → 蒸馏 → 事实
                   ↑
              唯一改动点
```

**实施内容**:
- 修改 `DistillerAgent` 的截断阈值
- 添加智能截断（语义边界）
- 添加配置项

**预估工作量**: 1小时

---

### 1.2 方案B: 智能内容处理

**来源**: `CONTENT_PROCESSOR_DESIGN.md`

**核心思路**: 用智能处理替代截断，包含三大模块

```
当前流程:
  抓取 → 清洗 → 截断 → 蒸馏 → 事实

优化后:
  抓取 → 结构化清洗 → 语义分块+筛选 → 跨源对齐 → 蒸馏 → 事实
          ↓              ↓              ↓
     元数据提取      相关性筛选      话题归类
```

**实施内容**:
- 新增 `ContentTransformer` 模块
- 新增 `SemanticChunker` 模块
- 新增 `CrossSourceAligner` 模块
- 重构 `graph.py` 数据流

**预估工作量**: 22小时

## 4. 优化建议

### 4.1 推荐方案: 动态阈值 + 智能截断

基于测试数据，推荐以下策略：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         推荐的截断策略                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  策略: 基于文档长度的动态阈值                                                │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  清洗后长度 ≤ 50,000字符                                             │    │
│  │  → 不截断，直接处理                                                  │    │
│  │  → 覆盖: 20% 的文档                                                  │    │
│  │  → 成本: 基准的 6x 以内                                              │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  50,000字符 < 清洗后长度 ≤ 150,000字符                               │    │
│  │  → 智能截断至 150,000字符 (语义边界)                                 │    │
│  │  → 覆盖: 60% 的文档                                                  │    │

│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  清洗后长度 > 150,000字符                                            │    │

│  │  → 覆盖: 20% 的文档                                                  │    │
│  │  → 可选: 分块处理 (用户配置)                                         │    │
```

### 4.2 具体实现建议

```python
class DistillerAgent:
    MAX_CONTENT_LENGTH = 150000  # 提高到150,000字符
    WARNING_THRESHOLD = 150000   # 超过此长度输出警告
    
    def _prepare_content(self, markdown_text: str) -> str:
        length = len(markdown_text)
        
        if length <= self.MAX_CONTENT_LENGTH:
            return markdown_text
        
        # 智能截断
        truncated = self._smart_truncate(markdown_text, self.MAX_CONTENT_LENGTH)
        
        if length > self.WARNING_THRESHOLD:
            print(f"[DistillerAgent] Warning: Content truncated from {length} to {self.MAX_CONTENT_LENGTH} chars")
        
        return truncated
    
    def _smart_truncate(self, text: str, max_length: int) -> str:
        """在语义边界截断"""
        if len(text) <= max_length:
            return text
        
        truncated = text[:max_length]
        
        # 优先在段落边界截断
        last_para = truncated.rfind('\n\n')
        if last_para > max_length * 0.8:
            return truncated[:last_para]
        
        # 其次在句子边界截断
        for end_char in ['。', '.', '！', '!', '？', '?']:
            last_sent = truncated.rfind(end_char)
            if last_sent > max_length * 0.8:
                return truncated[:last_sent + 1]
        
        return truncated
```

## 5. 结论

### 5.1 核心发现

1. **当前截断策略严重不足**: 8,000字符阈值导致100%的测试数据被截断，平均丢失94%的内容
2. **深度调研场景需要更长内容**: 测试样本平均长度137,995字符，远高于行业平均
3. **清洗阶段效率很高**: 0.21秒的平均清洗时间，降噪率45.5%，效果显著

### 5.2 推荐方案

**将截断阈值从8,000字符提高到150,000字符**，配合智能截断策略，可覆盖80%的文档，保留几乎全部信息。

### 5.3 成本考量

虽然API成本增加约19倍，但考虑到：
- 深度调研场景对信息完整性的高要求
- 用户愿意为高质量报告支付成本
- 可通过配置项灵活控制

**结论: 优化方案合理且必要。**
